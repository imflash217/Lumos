********-------------- NOTES on C++ ------------------------
01. If we subtract a value from 'unsigned' we must be sure that the result CANNOT be negative; 
   otherwise the resukt will be wraped.
02. Literal = 'fixed value in source code'
03. 'Integer Literals' can be written in decimal, octal, hexadecimal notation.
04. Integer literals that begin with 0 are interpreted as OCTAL (eg. 024)
05. Integer literals that begin with 0x, 0X are interpreted as HEXADECIMAL (eg. 0x24)
06. 'Decimal-literals' are SIGNED.
07. Octal/Hexadecimal-literals can either be SIGNED or UNSIGNED.
08. There are no literals of type 'short'.
09. Floating-point literals include either a decimal point or exponent specified using Scientific-Notation.
10. By Default; floating-point literals have type 'double'.
11. 'a' = character-literal.
12. "Hello World" = string-literal.
13. The words 'true' and 'false' are literals of type 'bool'.
14. The word 'nullptr' is a pointer literal.
15. 'Initialization' and 'Assignment' are different operations in C++.
    Initialization happens when a variable is given a value when it is created.
    Assignment destroys an object's current value and replaces it with the new one.
16. Variables defined outside any function body are initialized to 'zero'.
17. Variables defined inside a function body are 'uninitialized';(with one exception).
18. The value of an uninitialized variable is 'undefined'.
19. It is recommended to initialize object of every built-in type.
20. To obtain a 'declaration' that is NOT a 'definition', we add 'extern' keyword
      extern int a;  // declares but doesn't define.
      int a          // declares and defines.
      
      If we initialize a name and also add extern keyword; then the 'extern' is overwritten.
      extern int a = 99; // declares and defines.
      
      ERROR: Donot initialize on an 'extern' keyword.

21. Variables MUST be 'Defined' exactly ONCE; but can be 'Declared' many times.
22. C++ is a 'statically-typed' language i.e. the 'type' are checked at the COMPILE TIME.
23. In C++; the COMPILER checks whether the operation we write are supported by the 'type' or not.

24. COMPOUND TYPE: A 'type' defined in terms of another 'type'. For eg: references & pointers
25. REFERENCES: When we create a reference, we blind the reference to its initializer, insted of copying its the value of initializer.
      
   A 'reference' is NOT an 'object'; its just another name of an already existing 'object'.
   Because references are NOT objects, so we cannot DEFINE a reference with another reference.
   Each 'identifier' that is a reference must be preceded by '&' symbol.
